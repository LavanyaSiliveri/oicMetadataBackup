# oicMetadataBackup — OCI Function

An OCI Function that performs a scheduled metadata backup of an **Oracle Integration Cloud (OIC)** instance. On each run it exports all activated integrations as `.iar` archive files, captures connection and lookup metadata as JSON, and stores everything in an OCI Object Storage bucket under a timestamped prefix. Success and failure alerts are published to an OCI Notification Service (ONS) topic.

---

## What gets backed up

| Artifact | Format | OIC API used |
|---|---|---|
| Integrations | `.iar` binary archive | `GET /ic/api/integration/v1/integrations/{id}/archive` |
| Connection metadata | `connections.json` | `GET /ic/api/integration/v1/connections` |
| Lookup definitions | `lookups.json` | `GET /ic/api/integration/v1/lookups` |
| Package list | `packages.json` | `GET /ic/api/integration/v1/packages` |
| Run summary | `summary.json` | n/a — generated by the function |

> **Note:** Connection credentials are never exported. Only non-sensitive connection metadata (adapter type, name, status, etc.) is captured.

---

## Backup layout in Object Storage

```
{BACKUP_PREFIX}/
└── {YYYY-MM-DD_HH-MM-SS}/
    ├── integrations/
    │   ├── MY_INTEGRATION_01.00.0000.iar
    │   └── ANOTHER_FLOW_01.00.0000.iar
    ├── connections/
    │   └── connections.json
    ├── lookups/
    │   └── lookups.json
    ├── packages/
    │   └── packages.json
    └── summary.json
```

## Architecture

```
OCI Scheduler / cron / manual invoke
          │
          ▼
   OCI Functions (oicmetadatabackup)
          │
          ├── Reads OIC credentials from OCI Vault (recommended) or function config
          │
          ├── Calls OIC REST API (basic auth)
          │   ├── List integrations
          │   ├── Export each integration as .iar
          │   ├── List connections (metadata only)
          │   ├── List lookups
          │   └── List packages
          │
          ├── Uploads all artifacts to OCI Object Storage
          │
          └── Publishes success / failure notification to ONS
                    │
                    └── Email to subscribers
```

## Configuration reference

All settings are passed as OCI Function application config variables. Terraform sets them automatically; for manual setup use the OCI Console or `fn config app`.

| Config Key | Required | Default | Description |
|---|---|---|---|
| `OIC_BASE_URL` | Yes | — | OIC instance URL, e.g. `https://myoic.integration.ocp.oraclecloud.com` |
| `OIC_SERVICE_INSTANCE` | Yes | — | Service instance name shown on the OIC **About** page |
| `OIC_USERNAME` | * | — | OIC username (use Vault secret instead for production) |
| `OIC_PASSWORD` | * | — | OIC password (use Vault secret instead for production) |
| `OIC_USERNAME_SECRET_OCID` | * | — | OCI Vault secret OCID for OIC username — takes priority over `OIC_USERNAME` |
| `OIC_PASSWORD_SECRET_OCID` | * | — | OCI Vault secret OCID for OIC password — takes priority over `OIC_PASSWORD` |
| `OBJECT_STORAGE_BUCKET_NAME` | Yes | — | OCI Object Storage bucket name |
| `OBJECT_STORAGE_NAMESPACE` | No | auto-detected | Tenancy Object Storage namespace |
| `BACKUP_PREFIX` | No | `backups` | Top-level folder prefix in the bucket |
| `INCLUDE_INACTIVE` | No | `false` | Set to `true` to also back up CONFIGURED/DRAFT integrations |
| `BACKUP_CONNECTIONS` | No | `true` | Set to `false` to skip connection metadata export |
| `BACKUP_LOOKUPS` | No | `true` | Set to `false` to skip lookup export |
| `BACKUP_PACKAGES` | No | `true` | Set to `false` to skip package list export |
| `NOTIFICATION_TOPIC_OCID` | No | — | ONS topic OCID for success/failure email alerts |

\* Provide **either** the plain-text pair (`OIC_USERNAME` + `OIC_PASSWORD`) **or** the Vault OCID pair (`OIC_USERNAME_SECRET_OCID` + `OIC_PASSWORD_SECRET_OCID`). Vault takes priority when both are present.

## Terraform — Automated Infrastructure Provisioning

The `terraform/` directory provisions all required OCI infrastructure.

### What it provisions

| File | Resources created |
|---|---|
| `iam.tf` | Dynamic Group for the Function · IAM policy (Object Storage, Vault, ONS) |
| `object_storage.tf` | OCI Object Storage bucket for backups |
| `notifications.tf` | ONS Notification Topic · Email subscription |
| `functions.tf` | OCI Functions Application with all config vars pre-populated |

### Prerequisites

- Terraform >= 1.3.0
- OCI API key configured for the user running Terraform

### Usage

**1. Create a `terraform.tfvars` file** (excluded from git by `.gitignore`):

```hcl
tenancy_ocid         = "ocid1.tenancy.oc1..."
user_ocid            = "ocid1.user.oc1..."
fingerprint          = "aa:bb:cc:..."
private_key_path     = "~/.oci/oci_api_key.pem"
region               = "ap-sydney-1"
compartment_ocid     = "ocid1.compartment.oc1..."
subnet_ids           = ["ocid1.subnet.oc1..."]

oic_base_url         = "https://myoic.integration.ocp.oraclecloud.com"
oic_service_instance = "myoic"
notification_email   = "you@example.com"

# Option A — OCI Vault (recommended for production)
oic_username_secret_ocid = "ocid1.vaultsecret.oc1..."
oic_password_secret_ocid = "ocid1.vaultsecret.oc1..."

# Option B — direct credentials (development/testing only)
# oic_username = "oic.admin@example.com"
# oic_password = "MyPassword123"

# Optional overrides
# bucket_name          = "my-oic-backups"
# backup_prefix        = "backups"
# prefix               = "oicbackup"
# function_app_name    = "OICBackupFuncApp"
```

**2. Apply the Terraform:**

```bash
cd terraform
terraform init
terraform plan
terraform apply
```

**3. Deploy the function:**

```bash
cd ..
fn deploy --app OICBackupFuncApp
```

**4. Test the function:**

```bash
echo '{}' | fn invoke OICBackupFuncApp oicmetadatabackup
```

## Manual Setup (without Terraform)

### 1. Create an OCI Object Storage Bucket

1. Go to **Storage → Object Storage → Buckets → Create Bucket**
2. Set visibility to **Private**
3. Note the bucket name and namespace

### 2. Create an OCI Notification Topic

1. Go to **Developer Services → Application Integration → Notifications → Create Topic**
2. Click the topic → **Create Subscription** → Protocol: **Email**
3. Confirm the subscription email before notifications are delivered

### 3. Create an OCI Vault and Secrets (recommended)

1. Go to **Identity & Security → Vault → Create Vault**
2. Create two secrets: one for the OIC username, one for the OIC password
3. Note the OCID of each secret

### 4. Create a Dynamic Group for the Function

In **Identity → Dynamic Groups**, create a rule:

```
resource.type = 'fnfunc' AND resource.compartment.id = '<compartment-ocid>'
```

### 5. Create IAM Policies

```
Allow dynamic-group <dg-name> to manage objects in compartment <compartment-name> where target.bucket.name = '<bucket-name>'
Allow dynamic-group <dg-name> to read secret-bundles in compartment <compartment-name>
Allow dynamic-group <dg-name> to use ons-topics in compartment <compartment-name>
```

## Invoke

### Run a backup immediately (no body required — uses app config)

```bash
echo '{}' | fn invoke OICBackupFuncApp oicmetadatabackup
```

### Run with a config override (e.g. custom backup prefix)

```bash
echo '{
  "BACKUP_PREFIX": "manual-backup-2026-03"
}' | fn invoke OICBackupFuncApp oicmetadatabackup
```

### Include inactive integrations in this run

```bash
echo '{
  "INCLUDE_INACTIVE": "true"
}' | fn invoke OICBackupFuncApp oicmetadatabackup
```

## Scheduling

OCI Functions do not have a built-in scheduled trigger. Choose whichever of the following fits your environment:

### Option A — OCI DevOps Build Pipeline (recommended)

1. Create a **DevOps Project** and a **Build Pipeline** with a single shell build stage that runs:
   ```bash
   echo '{}' | fn invoke OICBackupFuncApp oicmetadatabackup
   ```
2. Add a **Trigger** of type **Schedule** to the Build Pipeline with the desired cron expression.

### Option B — Compute Instance cron job

On any OCI Compute instance with `fn` CLI configured:

```cron
# /etc/cron.d/oic-backup — run daily at 01:00 UTC
0 1 * * * opc /usr/local/bin/fn invoke OICBackupFuncApp oicmetadatabackup <<< '{}' >> /var/log/oic-backup.log 2>&1
```

### Option C — OCI Resource Scheduler (indirect)

Use OCI Resource Scheduler to start a Compute instance at the desired time, which in turn runs the function via a startup script, then stops itself.

```json
  "oic_base_url": "https://myoic.integration.ocp.oraclecloud.com",
  "service_instance": "myoic",
  "bucket": "oic-metadata-backups",
  "prefix": "backups/2026-03-05_01-00-00",
  "integrations": {
    "total": 42,
    "succeeded": 42,
    "failed": 0,
    "errors": []
  },
  "connections": { "total": 15, "backed_up": true },
  "lookups": { "total": 8, "backed_up": true },
  "packages": { "total": 5, "backed_up": true }

```

### Notification triggers

| Scenario | Notification title |
|---|---|
| All integrations backed up successfully | `OIC Backup - Completed Successfully` |
| One or more integrations failed | `OIC Backup - Completed with Errors` |
| Integration list API call failed | `OIC Backup - Integration List Failed` |

---

## Authentication

The function uses **Resource Principal** authentication for all OCI SDK calls (Object Storage, Vault, ONS) — no OCI credentials need to be embedded. For OIC itself, basic authentication (username/password) is used; credentials are retrieved from OCI Vault at runtime when Vault secret OCIDs are configured.
